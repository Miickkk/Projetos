MVC



Definição

É um tipo de Arquitetura de Software (modo como o código é organizado) que divide o projeto em camadas definidas, sendo elas o Model (manipulação de dados), View (interação com o usuário) e Controller (camada intermediária). É utilizado em aplicações mobile e web e em diferentes linguagens de programação como: Java, JavaScript, C#, PHP e Ruby, e em alguns frameworks como o Laravel e Spring, por exemplo.


Fundamentos:



Model:

Acesso e manipulação dos dados da aplicação. Responsável pelo gerenciamento de dados e estado da aplicação, realização de cálculos e operações de lógica, notificação de alterações ao Controller. É geralmente associado a linguagens de programação orientada a objeto (estrutura em classes), porém pode ser elaborado com outras estruturas como funções e objetos literais.

View:

Parte da interface do usuário, apresentando as informações e dados do Model de forma visual para que o usuário entenda. Também faz a coleta de entrada do usuário e recebe a ação para a interação do Model por meio do Controller.

Controller:

Camada de controle responsável por ligar o Model e a View, gerenciar o fluxo da aplicação e a comunicação entre o Model e View (camada intermediária), receber a entrada do usuário pelo View e traduzir em ações para o Model, controlar o fluxo da aplicação, determinando qual View deve ser exibida e atualizar o View quando o Model muda e vice-versa.

CRUD



Definição

CRUD é um acrônimo que representa as operações básicas em bancos de dados: Create (Criar), Read (Ler), Update (Atualizar) e Delete (Apagar). Essas operações são essenciais para manipular dados em sistemas de banco de dados e são fundamentais para quem deseja entender programação e gerenciamento de dados.


O que é CRUD?

São quatro ações que formam o cerne da maioria dos softwares de banco de dados, permitindo aos usuários criar registros novos, ler ou recuperar dados existentes, atualizar informações previamente armazenadas e, finalmente, excluir registros quando necessário.

O conceito de CRUD surgiu com os primeiros sistemas de gerenciamento de banco de dados. À medida que esses sistemas evoluíam, a necessidade de operações padrão para interagir com dados tornou-se evidente. O CRUD, portanto, representa essas operações essenciais, servindo como um framework básico para a interação de dados em inúmeras aplicações, oferecendo um modelo simples e eficiente para a manipulação de informações.


Fundamentos:



Criar/ Create (POST):

Os recursos são criados enviando solicitações HTTP POST para a API. O tipo de recurso é determinado pela URL da solicitação. O corpo da solicitação deve conter um objeto JSON que descreve o recurso a ser criado. O objeto no corpo da solicitação determina que o estado inicial do recurso será quando ele for criado. Alguns recursos exigem que determinadas propriedades sejam fornecidas quando são criadas, outras podem ser criadas com um objeto JSON vazio.

Em resumo, o POST é usado para criar recursos em um sistema, sendo importante fornecer as informações necessárias no corpo da solicitação para garantir a criação correta do recurso e suas relações.

Leitura/ Read (GET):

A leitura de recursos em uma API é feita através de solicitações HTTP GET. Existem duas formas principais de recuperar informações O primeiro método envolve a solicitação de uma lista de recursos, o segundo método é quando um único recurso é solicitado. Essa função garante que os dados armazenados possam ser recuperados e visualizados quando necessário.

Recuperação de múltiplos recursos:

Ocorre quando se solicita um ponto de extremidade sem especificar um recurso específico. Pode ser necessário incluir parâmetros de consulta para obter dados relevantes.

Exemplo: Para listar aplicativos IIS, é preciso informar o site específico: /api/webserver/webapps?website.id={website_id}.

Recuperação de um recurso específico

Acontece ao incluir a ID do recurso na URL do ponto de extremidade. Algumas APIs permitem o uso de parâmetros de consulta em vez da ID na URL.

Exemplo para recuperar um arquivo: Usando a ID: /api/files/{id} Usando o caminho físico: /api/files?physical_path={caminho} Isso permite flexibilidade na recuperação de dados, dependendo da estrutura da API.

Atualização/ Update (PATCH/PUT):

A atualização de dados em uma API pode ser feita através dos métodos PATCH e PUT, que servem para modificar informações já existentes.

PATCH: Atualiza parcialmente um recurso, alterando apenas os campos informados.

PUT: Atualiza um recurso inteiro, substituindo os dados existentes.

Ambos são essenciais para manter a precisão das informações em sistemas e bancos de dados.

Excluir/ Delete (DELETE):

O método DELETE é utilizado para remover um recurso de um sistema. A exclusão de um recurso ocorre ao enviar uma solicitação HTTP DELETE para a URL correspondente, que contém a ID do recurso. Essa operação deve ser usada com cautela, pois, dependendo da implementação, os dados podem ser permanentemente removidos e não recuperáveis.

CRUD em SQL

Para entender o CRUD no contexto do SQL, é vital conhecer como as operações são implementadas nesta linguagem. No SQL, a função Create é realizada através do comando “INSERT INTO”, que adiciona novos registros a uma tabela.

Read é feito usando “SELECT”, que recupera dados. “UPDATE” é usado para modificar registros existentes, e “DELETE” remove registros de uma tabela. Esses comandos são a base para interagir com dados em qualquer banco de dados SQL.

CRUD em Python

Em Python, o CRUD é aplicado frequentemente em combinação com bibliotecas como SQLAlchemy ou o módulo sqlite3 para interagir com bancos de dados. Por exemplo, para criar dados, você pode usar métodos para inserir objetos em uma tabela.

A leitura de dados é feita por meio de consultas, enquanto a atualização e a exclusão de dados são realizadas por meio de comandos específicos que modificam ou removem os objetos correspondentes.
